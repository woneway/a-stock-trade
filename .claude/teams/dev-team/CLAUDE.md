# A股交易系统开发团队

## 项目背景
A股交易系统 - 游资操盘辅助工具

## 技术栈
- **前端**: React 18 + TypeScript + Vite
- **后端**: Python FastAPI + SQLModel + SQLite
- **端口**: 前端 5173，后端 8001

## 团队成员

| 角色 | 职责 |
|------|------|
| PM | 项目管理、任务分配、进度追踪 |
| 产品 | 需求分析、功能设计、验收标准 |
| 前端 | React 页面开发、API 对接 |
| 后端 | FastAPI 接口、数据库设计 |
| QA | 功能验证、Bug 发现 |
| 用户 | 需求输入、体验反馈 |

## 自动化工作流程

```
体验 → 反馈 → PM整理 → 产品设计 → 开发实现 → QA验证 → 产品验收 → 反思改进 → 自动下一轮
```

### 自动循环流程
1. 使用 Playwright 自动体验所有页面
2. 收集 console 错误和 UI 问题
3. PM 整理反馈给产品
4. 产品设计解决方案
5. 前后端开发修复
6. QA 测试验证
7. 产品验收
8. **团队反思**：识别流程问题并改进
9. 自动进入下一轮体验

### 反思机制
每轮结束后自动反思：
- 团队协作问题
- 流程优化点
- 需要改进的 CLAUDE.md 配置

## 协作规范

1. **PM** 负责协调，使用 TodoWrite 追踪任务
2. **产品** 制定验收标准，各角色需遵循
3. **前端/后端** 需保持字段对齐
4. **QA** 严格按照验收标准测试
5. **用户** 反馈需具体场景
