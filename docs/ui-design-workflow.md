## 一、页面结构总览

```

│  整体页面结构                                                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 今日关注卡片设计

| 字段 | 数据来源 | 说明 |
|------|---------|------|
| 情绪周期 | 昨日复盘记录的market_cycle | 发酵期/启动期/高潮等 |
| 仓位建议 | 昨日复盘的position_advice | 如50-70% |
| 大盘位置 | 昨日复盘的market_analysis | 20日线上/下 |
| 热门板块 | 昨日复盘的hot_sectors | 板块列表 |
| 关注股票 | 昨日复盘的watch_stocks | 股票列表 |
| 风险提示 | 昨日复盘的risk_warning | 风险描述 |

## 七、交互流程设计

### 7.1 盘前流程

```
1. 用户进入"今日计划"页面
2. 系统自动加载最新复盘记录
3. 显示"昨日复盘回顾"卡片
4. 显示"今日关注"提醒（基于昨日复盘）
5. 用户可以：
   a. 编辑今日市场分析数据
   b. 添加/编辑/删除关注股票
   c. 设置买入条件
   d. 保存复盘
6. 点击"确认计划"锁定今日计划
```

### 7.2 盘中流程

```
1. 盘中Tab显示候选股票列表
2. 每个候选股票显示：
   - 当前价格和距离目标价
   - 验证条件状态
   - 买入/放弃按钮
3. 点击"买入"弹出买入弹窗
   - 显示买入条件检查清单
   - 填写实际成交信息
   - 设置止损止盈
4. 点击"卖出"弹出卖出弹窗
   - 显示触发条件
   - 显示其他卖出信号
   - 选择卖出数量和原因
5. 实时显示今日成交记录
```

### 7.3 盘后流程

```
1. 盘后Tab显示：
   - 今日交易汇总
   - 计划vs实际对比
2. 用户填写复盘：
   - 情绪记录
   - 正确的地方
   - 错误的地方
   - 明日改进
3. 可预填明日计划
4. 保存复盘（成为明日的"昨日复盘"）
```

## 八、总结

| 页面 | 核心功能 | 数据来源 |
|------|---------|---------|
| 首页 | 今日概览+今日关注 | 来自最新复盘记录 |
| 今日计划-盘前 | 复盘+计划 | 复盘记录+候选股票 |
| 今日计划-盘中 | 执行+记录 | 候选股票+交易记录 |
| 今日计划-盘后 | 复盘总结 | 交易记录+复盘内容 |

**核心原则**：
- 复盘数据驱动今日关注
- 计划指导盘中执行
- 执行反馈完善复盘
- 形成闭环
