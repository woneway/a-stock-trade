# 前后端文件沟通协议

## 沟通流程

1. **发起请求**: 前端/后端在 `requests/` 目录创建请求文件
2. **处理请求**: 接收方读取请求并处理
3. **响应请求**: 在 `responses/` 目录创建响应文件
4. **状态更新**: 请求文件状态更新为 completed

## 文件命名规则

```
请求:  request_{序号}_{发起方}_{主题}.md
响应:  response_{序号}_{接收方}_{主题}.md
```

## 请求文件格式

```markdown
# API 请求 #001 - [主题]

## 发起方
前端 / backend

## 优先级
高 / 中 / 低

## 需求描述
[详细描述需要实现的功能或修复的问题]

## 具体内容
### API 路径
POST /api/xxx

### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| xxx    | string | 是   | xxx  |

### 期望响应
```json
{
  "code": 200,
  "data": {}
}
```

## 状态
pending / in_progress / completed

## 创建时间
2026-02-22 10:00:00
```

## 响应文件格式

```markdown
# API 响应 #001 - [主题]

## 关联请求
request_001_frontend_xxx.md

## 处理状态
completed / rejected / needs_discussion

## 实现说明
[详细说明实现方式]

## API 实现
### 路由
POST /api/xxx

### 参数处理
[参数校验逻辑]

### 返回格式
```json
{
  "code": 200,
  "data": {},
  "message": "success"
}
```

## 备注
[其他说明]

## 完成时间
2026-02-22 10:30:00
```

## 使用方法

1. 我(AI) 读取请求文件
2. 判断是前端需求还是后端需求
3. 将请求转发给对应的接收方
4. 接收方实现后创建响应文件
5. 我读取响应并继续协调
