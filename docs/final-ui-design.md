## 七、数据模型设计

### 7.1 渐进式设计

```
第一阶段（兼容现有系统）：
- 复用现有字段
- 只增加关键字段：market_cycle, position_advice, risk_warning

第二阶段（功能扩展）：
- 增加板块相关字段
- 增加大盘位置字段

第三阶段（完整功能）：
- 增加统计类字段
- 支持自动化数据获取
```

### 7.2 复盘记录（第一阶段）

```typescript
interface Review {
  // 必填
  id: number;
  trade_date: string;
  limit_up_count: number;
  limit_down_count: number;
  market_cycle: string;
  position_advice: string;
  hot_sectors: string[];

  // 选填
  broken_plate_ratio?: number;
  highest_board?: number;
  up_count?: number;
  turnover?: number;
  risk_warning?: string;

  // 扩展（第二阶段）
  above_20ma?: boolean;
  index_trend?: string;
  main_sectors?: string[];
  sector_stocks?: string[];
}
```

### 7.3 交易记录

```typescript
interface Trade {
  // 现有字段
  id: number;
  stock_code: string;
```

## 九、设计原则总结

| 原则 | 说明 |
|------|------|
| 简洁优先 | 盘前信息精简，突出重点 |
| 快速操作 | 买入/卖出弹窗简化，一键确认 |
| 关键提醒 | 时间止损等核心功能强提醒 |
| 渐进扩展 | 数据模型分阶段实现 |
| 兼容现有 | 字段名避免冲突，复用现有能力 |
